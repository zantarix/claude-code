# Preserve observable multi-phase contracts over atomic shortcuts

When a single-call API would collapse an existing multi-phase contract, default to preserving the contract's observable phasing rather than taking the atomic shortcut. Before replacing a multi-phase implementation with a single atomic call, check whether any caller or workflow depends on an intermediate phase being observable (e.g. a record/stage step distinct from a publish/commit step). If so, keep the phases separate — materialise side effects at the phase the contract expects — and raise the trade-off to the user instead of silently collapsing.
