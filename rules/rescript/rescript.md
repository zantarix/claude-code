# ReScript conventions

- Source files use `.res` (implementation) and `.resi` (interface/public API) extensions. Compiled output is `.res.mjs`, generated in-source alongside `.res` files — gitignored, must be regenerated by the ReScript compiler.
- ReScript has a flat module namespace. Subdirectories are purely organisational and do not affect module names or imports.
- Third-party JavaScript libraries require explicit FFI bindings. There is no implicit interop.
- ReScript cannot express React class components. Any component requiring class lifecycle methods (e.g. error boundaries) must be written in JavaScript and bound via FFI.
